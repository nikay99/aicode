# AICode v0.4 Roadmap
## Production-Ready Package Manager & Developer Experience

**Target Release**: Q2 2026  
**Duration**: 8-10 Wochen  
**Team Size**: 2-3 Entwickler  
**Status**: v0.3.1 Production Ready → v0.4 Ecosystem Ready

---

## Executive Summary

AICode v0.4 transformiert die Sprache von einem "funktionierenden Compiler" zu einem produktionsreifen Ökosystem. Der Fokus liegt auf:

1. **Package Manager** - Ermöglicht Code-Sharing und Wiederverwendung
2. **LSP & IDE Support** - Professionelle Developer Experience
3. **Tooling** - Formatter, Linter, Debugger für Produktivität

## Phase 1: Foundation (Woche 1-2)

### Week 1: Package Manager - Core Infrastructure
**Goal**: Lokale Paketverwaltung funktioniert

**Tasks**:
- [ ] `src/package_manager.py` - Core Klasse (3 Tage)
  - Paket-Metadaten parsing (aicode.toml)
  - Lokale Installation in `~/.aic/packages/`
  - Lockfile-Generierung (aic.lock)
- [ ] `aic install <package>` CLI Command (1 Tag)
- [ ] `aic install` (aus aicode.toml) (1 Tag)

**Abhängigkeiten**: Keine  
**Parallelisierbar**: Ja

### Week 2: Package Manager - Dependency Resolution
**Goal**: Transitive Dependencies & Semver

**Tasks**:
- [ ] Semver parsing & comparison (1 Tag)
- [ ] Dependency Graph Resolution (2 Tage)
  - Topological sort für Installationsreihenfolge
  - Konflikt-Erkennung (gleiche Paket, verschiedene Versionen)
- [ ] Lokale Registry-Index (1 Tag)
  - `~/.aic/registry/index.json`
  - Paket-Suche & Metadaten-Caching
- [ ] `aic list` & `aic remove` (1 Tag)

**Abhängigkeiten**: Week 1  
**Blocker**: Nein

---

## Phase 2: LSP Foundation (Woche 3-4)

### Week 3: Language Server Protocol - Core
**Goal**: LSP Server startet und kommuniziert

**Tasks**:
- [ ] `lsp/server.py` - JSON-RPC Server (2 Tage)
  -stdio Transport
  - Message parsing & dispatch
  - Lifecycle management (initialize/shutdown)
- [ ] `lsp/handlers.py` - Request Handler (2 Tage)
  - textDocument/completion
  - textDocument/hover
  - textDocument/definition
- [ ] `aic lsp` CLI Command (1 Tag)

**Abhängigkeiten**: Keine  
**Parallelisierbar**: Ja

### Week 4: LSP Features - Intelligence
**Goal**: Autocomplete & Hover funktionieren

**Tasks**:
- [ ] Symbol Table Builder (2 Tage)
  - Aus AST exports/imports extrahieren
  - Scope-Analyse für lokale Variablen
- [ ] Autocomplete Provider (2 Tage)
  - Kontext-basierte Vorschläge
  - Snippets für häufige Patterns
- [ ] Hover Information (1 Tag)
  - Type info aus Type Checker
  - Docstrings (wenn vorhanden)

**Abhängigkeiten**: Week 3, Type Checker  
**Blocker**: Nein

## Risiken & Mitigation

| Risiko | Wahrscheinlichkeit | Impact | Mitigation |
|--------|-------------------|--------|------------|
| LSP Komplexität | Mittel | Hoch | Früh mit einfachem Protokoll starten |
| Type Checker Performance | Niedrig | Hoch | Caching, inkrementelle Updates |
| Package Manager Edge Cases | Mittel | Mittel | Extensive Tests, Semver-Compliance |
| VS Code Review Zeit | Hoch | Mittel | Früh einreichen, Beta-Channel |
