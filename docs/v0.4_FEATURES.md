# AICode v0.4 - Package Manager & Developer Tools

## Overview

AICode v0.4 introduces a complete ecosystem for professional development:

- **Package Manager** - Install and manage AICode packages
- **LSP Server** - Language Server Protocol for IDE integration
- **VS Code Extension** - Full IDE support
- **Formatter v2** - AST-based code formatting
- **Linter v1** - Static code analysis

## Package Manager

### Installation

```bash
# Install from local registry
aic install <package>

# Install specific version
aic install <package>@1.0.0

# Install from local file/directory
aic install mypackage --from-file ./mypackage.tar

# Install all dependencies from aicode.toml
aic install
```

### Managing Packages

```bash
# List installed packages
aic list

# Remove a package
aic remove <package>
```

### Package Structure

Packages should include `aicode.toml`:

```toml
[project]
name = "mypackage"
version = "1.0.0"
description = "My AICode package"

[dependencies]
other_package = "^1.0.0"
```

## Language Server Protocol (LSP)

### Running the LSP Server

```bash
# Start the LSP server
aic lsp
```

The LSP server provides:
- **Autocomplete** - Context-aware suggestions
- **Hover** - Type information on hover
- **Go to Definition** - Navigate to symbol definitions
- **Diagnostics** - Real-time error highlighting

### VS Code Extension

1. Install from marketplace (search "AICode")
2. Open a `.aic` file
3. The extension automatically connects to LSP

Manual installation:
```bash
cd extension
npm install
code --install-extension ./
```

## Formatter v2

```bash
# Format a file (print to stdout)
aic format hello.aic

# Format in place
aic format hello.aic --in-place
```

The AST-based formatter produces consistent, idiomatic AICode.

## Linter v1

```bash
# Lint a file
aic lint hello.aic
```

Checks include:
- **W041** - Shadowing existing symbols
- **W042** - Function may not return value
- **W043** - Unused variables
- **W044** - Unreachable code after return
- **E040** - Assignment to undefined variable

## Directory Structure

```
~/.aic/
├── packages/           # Installed packages
│   └── <package>/
│       └── aicode.toml
├── registry/
│   └── index.json      # Local package index
└── aic.lock           # Lockfile
```

## Error Codes

### Package Manager (E3xx)
- **E301** - Package not found in registry
- **E302** - Dependency resolution failed
- **E303** - Invalid package metadata

### Linter (W04x/E04x)
- **W041** - Symbol shadowing warning
- **W042** - Missing return warning
- **W043** - Unused variable warning
- **W044** - Dead code warning
- **E040** - Undefined variable error

## Configuration

Create `aicode.toml` in your project:

```toml
[project]
name = "myproject"
version = "0.1.0"

[dependencies]
some_package = "^1.0.0"

[build]
entry = "main.aic"
```

## Examples

### Creating a New Project

```bash
aic init myproject
cd myproject
aic install
aic run main.aic
```

### Installing and Using a Package

```bash
# Create a project
aic init myapp
cd myapp

# Install dependencies
aic install

# Add a dependency
echo '
[dependencies]
utils = "^1.0.0"
' >> aicode.toml
aic install

# Run
aic run main.aic
```
